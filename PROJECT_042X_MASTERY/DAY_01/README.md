## Learning Objectives

By end of Day 1 you should be able to:

- [ ] Give a 90-second product pitch from memory
- [ ] Draw the high-level architecture on a whiteboard
- [ ] Name all 6 key features and explain each in 1 sentence
- [ ] Explain what a BFF architecture is and why it was chosen
- [ ] Walk through any file in the folder structure and explain why it exists
- [ ] Describe the complete data flow from user input to rendered graph

---

## Day 1 Homework

1. Open the actual codebase. Navigate every folder manually.
2. Run the project locally. Analyze a real repository.
3. Draw the architecture on paper (not a tool) — 3 times.
4. Practice the 90-second pitch out loud, timed.

---

## Interview Confidence Target

After Day 1: You should be at **40%** interview readiness.
You know *what* the project does. Days 2–7 teach you *how* and *why*.
